# 柴火创客 OPC 学院课程站改版设计

日期：2026-05-28
状态：设计已确认，等待用户复审

## 背景

当前站点已经有一批有效内容：M0-M5 课程模块、L1-L3 学习深度、推荐路径、合作场景、合作形态、柴火与 Seeed 的生态背景。主要问题不是“没有内容”，而是页面之间反复解释同一套信息，视觉表达也在新版 panel 语言与旧式卡片语言之间摇摆。

这次改版的目标不是重写品牌口号，而是重排页面职责，让用户一步一步看懂这套课程，并自然判断它与自己的学习、教学、项目、服务或合作需求的关系。

## 总领思路

柴火创客 OPC 学院负责的这套课程，应被表达为一张真实技术课程地图，而不是一组固定不变的课程清单。

当前 M0-M5 是这张地图的阶段性版本：从硬件基础出发，延伸到设备互联、远距物联、视觉 AI、空间智能和行业交付。未来随着技术发展，学院可以继续扩展新的系统性课程，因此页面不应把 M0-M5 写成学院的最终边界。

页面不替用户下结论，不把用户提前锁定为“负责人”“工程师”“企业客户”等单一身份。站点提供清晰材料和阅读路径，让不同用户自己完成投射。

## 用户认知路径

站点应形成一个课程认知漏斗：

1. 用户先看见熟悉的技术物件或问题。
2. 用户发现这些物件背后有一张更完整的技术课程地图。
3. 用户理解可以从不同方向、深度和目标进入。
4. 用户看到这些技能可以变成项目、服务、交付物或组织能力。
5. 用户再决定继续学习、选课、转述给团队，或发起合作咨询。

这条路径的核心原则是：先让用户在认知内找到入口，再展示超出原有认知的系统性和可交付价值。

## 页面架构

### 首页

首页负责建立兴趣和方向感，不展开完整矩阵。

首页应保留“从第一次点亮 LED，到独立交付智能系统”这类开放式能力跨度表达。它不限定用户身份，也不把课程窄化成某个单一合作对象。

首页主要内容：

- 用开放主标题建立从入门到交付的跨度。
- 从 LED、传感器、网关、摄像头、空间设备、交付文档等熟悉物件进入。
- 简要露出 M0-M5 和 L1-L3，但只解释“这张图怎么读”。
- 提供开放入口：从基础开始、从技术方向开始、从项目目标开始、从组织引入课程开始。
- 展示真实硬件、真实场景、真实交付，而不是空泛价值观。
- 以“查看课程地图”为主 CTA，合作咨询作为次级入口。

首页不承担完整课程说明，也不提前把用户分类。

### 课程体系页

课程体系页是主地图，负责讲清 M0-M5 x L1-L3。

页面应让用户明确：

- 横轴 M0-M5 表示当前课程方向。
- 纵轴 L1-L3 表示学习深度和阅读深度。
- M0 是基础起点，M1-M4 是可独立深入的技术方向，M5 是行业交付整合。
- L1/L2/L3 不只代表难度，也代表不同程度的理解、搭建和交付能力。
- 用户可以按方向、深度或目标组合，不必被迫从头到尾线性学习。

课程体系页应以矩阵和路径为主，卡片只作为解释辅助。

### 模块详情页

每个模块详情页负责把一个技术方向讲具体。

统一结构：

- 这个模块解决什么真实问题。
- 会接触哪些硬件、工具、平台和方法。
- L1、L2、L3 分别学习什么、产出什么。
- 学完后能形成哪些可用结果，例如 demo、POC、部署方案、客户演示、验收材料、交付文档、可复制方案。
- 适合与哪些模块组合。

模块详情页是“看起来能拿来做项目、做服务、形成收入”的关键页面。它不直接承诺赚钱，而是展示可服务、可项目化、可交付的产出物。

### 路径入口

路径入口可以先作为课程页或首页中的重点 section，未来可独立成页。

路径不按身份强分类，而按目标组织：

- 从硬件基础开始。
- 搭建一个可运行的智能空间。
- 做远距物联或传感网络。
- 把视觉 AI 放到边缘设备上。
- 把 demo 变成可交付项目。

这种组织方式允许高校、OPC、小型集成商、企业团队、工程师、老师和学生自行代入。

### 合作页

合作页负责把已经产生的理解转成行动。

合作页不从头重复课程体系，而是说明柴火创客 OPC 学院如何帮助组织引入、共建或定制课程。

主要内容：

- 高校、OPC、小型集成商、企业团队等不同合作语境。
- 标准课程授权、联合课程共建、企业定制内训、战略联合交付等合作形态。
- 当前课程地图如何被引入组织、课程、项目或服务体系。
- 清晰联系方式和表单。

合作页可以更明确地谈对象和合作方式，因为用户进入这里时已经接受被分类。

### 关于页

关于页负责可信度，不重复课程体系。

主要内容：

- 柴火创客空间的创客与教育背景。
- Seeed 的硬件生态和工程资源。
- 柴火创客 OPC 学院为什么适合承接这套课程。
- 真实硬件、真实项目和真实社区如何进入课程设计。

## 首页叙事顺序

首页应按以下顺序组织：

1. 首屏用开放主标语建立能力跨度，主 CTA 指向课程地图。
2. 用熟悉物件串起系统能力，展示从硬件到交付的自然演进。
3. 露出 M0-M5 与 L1-L3 的读图方式，不展开完整矩阵。
4. 提供几种开放读法，让用户按基础、方向、目标或合作需求进入。
5. 展示可落地能力结果，例如系统搭建、POC、调试、部署、文档、交付。
6. 最后再放可信度和合作入口。

整体节奏是：熟悉的起点 -> 更大的课程地图 -> 可选择的路径 -> 可落地的技能 -> 再考虑合作。

## 视觉语言

视觉方向应是“有现场感的技术课程地图”。

不走抽象科技感官网，也不走普通培训招生页。页面需要同时有工坊感、课程秩序感和技术交付可信度。

视觉原则：

- 使用真实硬件、工作台、线缆、网关、摄像头、学员操作等素材作为长期方向。
- M0-M5、L1-L3、路径箭头、模块编号形成视觉骨架。
- 黄色承担课程、路径和模块的温度；黑色承担总结、可信度和交付结果；红色只做少量引导。
- 减少“很多卡片”的堆叠，增加路径、矩阵、分层条、模块地图等图谱式结构。
- 文案版式更像课程手册：编号、短句、注释、路径说明清楚可转述。
- 商业价值通过产出物体现，不直接写“学完能赚钱”。

应避免：

- 过早用身份标签限制用户。
- 把 M0-M5 写成学院全部课程边界。
- 用封闭式品牌口号替用户下结论。
- 大量营销式 CTA 堆叠。
- 抽象科技渐变、过度装饰、普通 SaaS 卡片感。

## 内容结构原则

已有文字内容应按页面职责重新分配：

- 品牌和生态背景放在关于页与首页末段。
- 完整课程矩阵放在课程体系页。
- 模块细节和产出物放在模块详情页。
- 目标式学习路径放在首页或课程页的路径入口。
- 合作对象和合作形态放在合作页。

每段内容都应回答一个具体问题：

- 这是什么？
- 我怎么读懂它？
- 我从哪里进入？
- 学到什么深度？
- 能做出什么？
- 为什么可信？
- 下一步怎么联系？

## 成功标准

改版后应达到：

- 用户不需要反复阅读多个页面才能理解课程体系。
- 首页不再抢课程页和合作页的工作。
- 课程页成为清晰主地图。
- 模块页能让技术方向具体、可感、可项目化。
- 合作页承接理解后的行动，而不是重新营销。
- 视觉更像技术课程地图，有真实工坊感和体系感。
- 未来新增系统性课程时，首页叙事仍然成立。

## 非目标

本轮设计不做以下事情：

- 不重新发明品牌 slogan。
- 不把用户强行分类为单一身份。
- 不承诺“学完能赚钱”。
- 不引入与当前课程无关的大范围内容。
- 不为符合视觉比例而堆无意义色块。
- 不在设计阶段写代码或重构组件。
